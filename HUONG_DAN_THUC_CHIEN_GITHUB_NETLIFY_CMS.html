Hình 1: Luồng vận hành GitHub + Netlify. Netlify Functions xử lý API lưu nội dung.

2. File Netlify đã chuẩn bị trong dự án

2.1. netlify.toml

Dự án đã có file netlify.toml ở thư mục gốc:

[build]
  command = "node scripts/build.js"
  publish = "."
  functions = "netlify/functions"

[[redirects]]
  from = "/admin"
  to = "/admin/index.html"
  status = 200

[[redirects]]
  from = "/api/content"
  to = "/.netlify/functions/content"
  status = 200

[[redirects]]
  from = "/api/save"
  to = "/.netlify/functions/save"
  status = 200

2.2. Netlify Functions

Hai function đã được thêm:

  • netlify/functions/content.js: đọc dữ liệu CMS.
  • netlify/functions/save.js: lưu thay đổi bằng cách commit về GitHub.

Nhờ redirect trong netlify.toml, admin vẫn gọi /api/content và /api/save như bản Vercel.

2.3. Kiểm tra build local

cd "C:\Users\TPL\Downloads\web codex"
node scripts/build.js

3. Đưa dự án lên GitHub

3.1. Tạo repository

2

Đặt tên repo, ví dụ hong-nhung-bds.

Chọn Private nếu chưa muốn public mã nguồn.
5
Bấm Create repository.

Hình 2: Tạo repo trên GitHub.

3.2. Push bằng Git command line

Cách này ổn định hơn upload web, đặc biệt khi thư mục có ảnh/video lớn.

cd "C:\Users\TPL\Downloads\web codex"
git init
git add .
git commit -m "Initial Netlify CMS site"
git branch -M main
git remote add origin https://github.com/<OWNER>/<REPO>.git
git push -u origin main

Thay <OWNER> và <REPO> bằng thông tin repo của chị.

Nếu upload bằng giao diện web GitHub, file lớn khoảng trên 25 MiB có thể lỗi. Trong dự án hiện có file ảnh gốc khoảng 29 MB, nên push bằng Git command line là cách nên dùng.

4. Tạo GitHub token cho CMS

Token này để Netlify Function commit dữ liệu mới vào GitHub. Không đưa token vào file trong repo.

1
Vào https://github.com/settings/personal-access-tokens.
2
Chọn Fine-grained tokens → Generate new token.
3
Repository access: chọn đúng repository website.
4
Repository permissions → Contents: chọn Read and write.
5
Bấm Generate token, copy token ngay.

Hình 3: Token cần quyền Contents: Read and write.

7. Kết nối tên miền sunshine-legendcity.io.vn

7.1. Mục tiêu

Sau khi cấu hình xong, website sẽ mở được bằng:

  • https://sunshine-legendcity.io.vn
  • https://www.sunshine-legendcity.io.vn

Khuyến nghị thực tế: dùng sunshine-legendcity.io.vn làm domain chính, còn www.sunshine-legendcity.io.vn tự chuyển hướng về domain chính.

7.2. Thêm domain trong Netlify trước

1
Vào Netlify project của website.
2
Ở menu trái, chọn Domain management.
3
Bấm Add a domain hoặc Add custom domain.
4
Nhập chính xác: sunshine-legendcity.io.vn.
5
Xác nhận thêm domain. Netlify có thể tự thêm thêm bản www.sunshine-legendcity.io.vn.

Hình 7: Thêm custom domain trong Netlify trước khi chỉnh DNS.

7.3. Cách dễ kiểm soát nhất: giữ DNS tại nơi mua tên miền

Đăng nhập vào nơi chị mua/quản lý tên miền sunshine-legendcity.io.vn, mở phần DNS hoặc Quản lý bản ghi DNS. Thêm/sửa các bản ghi sau:

LoạiHost / NameValue / Points toGhi chú
A @ hoặc để trống 75.2.60.5 Trỏ domain gốc sunshine-legendcity.io.vn về Netlify.
CNAME www <ten-site-netlify>.netlify.app Thay bằng URL Netlify thật, ví dụ sunshine-legendcity.netlify.app.

Nếu nhà cung cấp DNS hỗ trợ ALIAS, ANAME hoặc flattened CNAME cho host @, có thể trỏ @ tới apex-loadbalancer.netlify.com thay cho bản ghi A. Nếu không thấy các loại bản ghi đó, dùng A record 75.2.60.5.

Hình 8: Bản ghi DNS cần tạo ở nơi quản lý tên miền.

7.4. Chọn Primary domain trong Netlify

1
Quay lại Netlify → Domain management.
2
Đợi domain hết trạng thái Pending DNS verification.
3
Ở sunshine-legendcity.io.vn, chọn menu tùy chọn → Set as primary domain.
4
Netlify sẽ chuyển hướng domain phụ về domain chính.

Nếu muốn dùng www.sunshine-legendcity.io.vn làm domain chính, đặt www làm primary. Tuy nhiên theo yêu cầu hiện tại, nên đặt domain gốc sunshine-legendcity.io.vn làm primary.

7.5. Bật HTTPS / SSL

1
Trong Netlify project, vào Domain management → HTTPS.
2
Chọn Verify DNS configuration nếu có nút này.
3
Chọn Provision certificate hoặc chờ Netlify tự cấp chứng chỉ Let's Encrypt.
4
Khi trạng thái HTTPS thành công, mở https://sunshine-legendcity.io.vn.

DNS có thể cần vài giờ, đôi khi 24-48 giờ, để lan truyền toàn mạng. Trong thời gian đó domain có thể lúc được lúc chưa.

7.6. Kiểm tra sau khi cấu hình

Kiểm tra bằng trình duyệt:

  • https://sunshine-legendcity.io.vn mở được website.
  • https://www.sunshine-legendcity.io.vn tự chuyển về domain chính hoặc cũng mở được website.
  • https://sunshine-legendcity.io.vn/admin mở được CMS.

Nếu có terminal, có thể kiểm tra DNS:

nslookup sunshine-legendcity.io.vn
nslookup www.sunshine-legendcity.io.vn

Kết quả mong muốn:

  • sunshine-legendcity.io.vn có A record về 75.2.60.5 nếu dùng external DNS.
  • www.sunshine-legendcity.io.vn có CNAME về site Netlify.

7.1. Mở trang quản trị

Hình 7: Đăng nhập /admin.

7.2. Quy trình sửa nội dung

  1. Chọn tab cần sửa: Thông tin chung, Trang chủ, Giới thiệu, Liên hệ hoặc Dự án.
  2. Sửa nội dung.
  3. Bấm Lưu & xuất bản.
  4. Netlify Function commit vào GitHub.
  5. Netlify nhận commit mới và deploy lại.
  6. Mở website, bấm Ctrl + F5 nếu còn cache cũ.

  • Netlify deploy thành công.
  • Netlify có đủ biến môi trường.

  • Sửa thử một nội dung nhỏ, bấm lưu, thấy GitHub có commit mới.
