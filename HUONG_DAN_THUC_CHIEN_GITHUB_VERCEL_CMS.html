1. Sơ đồ hoạt động

Hình 1: Luồng xuất bản nội dung. Người quản trị không sửa HTML trực tiếp nữa, mà sửa trong /admin.

2. Chuẩn bị trước khi upload

2.1. Kiểm tra build local

Mở PowerShell tại thư mục dự án rồi chạy:

cd "C:\Users\TPL\Downloads\web codex"
node scripts/build.js

Kết quả đúng sẽ có dòng tương tự:

Built 21 project pages from content JSON.

Không cần chạy npm. Nếu PowerShell chặn npm.ps1, vẫn dùng được node scripts/build.js.

2.2. Xem thử trên máy

node scripts/serve.js

Mở trình duyệt:

  • http://localhost:4173
  • http://localhost:4173/admin

Local server có API giả lập để test lưu JSON tại máy. Khi lên Vercel, API thật sẽ commit lên GitHub.

2.3. Cực kỳ quan trọng: file lớn khi upload GitHub

Trong thư mục hiện tại có một số file ảnh/video rất lớn. Nếu upload bằng giao diện web GitHub, file quá lớn có thể bị từ chối. GitHub Docs nêu rằng upload qua browser giới hạn 25 MiB mỗi file; dùng command line có thể đưa file lớn hơn, tối đa 100 MiB mỗi file.

File cần chú ý: Căn Hộ Sunshine Legend City.png khoảng 29 MB. Nếu upload bằng web GitHub, file này có khả năng lỗi.

Khuyến nghị thực chiến: dùng Git command line để push toàn bộ dự án. Nếu chị muốn upload bằng web GitHub, hãy bỏ các file gốc lớn không dùng khỏi lần upload hoặc nén ảnh trước.

3. Đưa dự án lên GitHub

3.1. Tạo repository mới

1
Vào https://github.com, đăng nhập tài khoản.
2
Bấm nút dấu + góc trên bên phải, chọn New repository.
3
Đặt tên repo, ví dụ hong-nhung-bds.
4
Chọn Private nếu chưa muốn công khai mã nguồn. Có thể dùng Public nếu không ngại public.
5
Không tick README, không tick .gitignore, không tick license nếu dùng lệnh bên dưới. Bấm Create repository.

Hình 2: Tạo repository mới trên GitHub.

3.2. Cách khuyến nghị: push bằng Git command line

Cài Git cho Windows nếu máy chưa có. Sau đó mở PowerShell tại thư mục dự án và chạy:

cd "C:\Users\TPL\Downloads\web codex"
git init
git add .
git commit -m "Initial CMS site"
git branch -M main
git remote add origin https://github.com/<OWNER>/<REPO>.git
git push -u origin main

Thay:

  • <OWNER> bằng tên tài khoản GitHub, ví dụ hongnhung.
  • <REPO> bằng tên repository, ví dụ hong-nhung-bds.

Vì sao nên dùng Git command line? Cách này xử lý thư mục nhiều file tốt hơn, tránh lỗi upload web với file lớn trên 25 MiB.

3.3. Cách phụ: upload bằng web GitHub

Chỉ dùng cách này nếu không cài Git.

1
Vào repository vừa tạo.
2
Bấm Add file → Upload files.
3
Kéo thả toàn bộ file/thư mục của dự án vào vùng upload.
4
Nếu báo lỗi file quá lớn, bỏ file đó ra, nén lại hoặc dùng Git command line.
5
Nhập commit message: Initial CMS site, bấm Commit changes.

Hình 3: Upload file bằng giao diện GitHub.

4. Tạo GitHub token cho CMS

Token này cho phép Vercel API ghi thay đổi vào repository. Chỉ đặt token trong biến môi trường Vercel, không dán vào file HTML/JS/JSON.

1
Vào https://github.com/settings/personal-access-tokens.
2
Chọn Fine-grained tokens → Generate new token.
3
Token name: vercel-cms-hong-nhung-bds.
4
Repository access: chọn Only selected repositories, chọn đúng repo website.
5
Permissions → Repository permissions → Contents: chọn Read and write.
6
Bấm Generate token, copy token ngay. GitHub chỉ hiện token một lần.

Hình 4: Token cần quyền Contents: Read and write với đúng repository.

5. Deploy bằng Vercel

5.1. Import GitHub repository

1
Vào https://vercel.com, đăng nhập.
2
Bấm Add New → Project.
3
Chọn GitHub, tìm repository hong-nhung-bds, bấm Import.
4
Framework Preset: Other.
5
Build Command: node scripts/build.js.
6
Output Directory: .. Dự án cũng đã có vercel.json để khai báo cấu hình.

Hình 5: Import repo vào Vercel và dùng build command của dự án.

5.2. Thêm biến môi trường

Trong màn hình import hoặc sau khi tạo project: vào Settings → Environment Variables và thêm đủ các biến sau.

Tên biếnGiá trị mẫuGhi chú
ADMIN_PASSWORDmat-khau-rieng-cua-chiMật khẩu đăng nhập /admin.
GITHUB_TOKENgithub_pat_...Token đã tạo ở bước 4.
GITHUB_OWNERten-tai-khoan-githubOwner repo, ví dụ tên tài khoản GitHub.

Sau khi thêm hoặc sửa Environment Variables, cần deploy lại. Vercel áp dụng biến mới cho deployment mới, không tự thay đổi deployment cũ.

Hình 6: Nhập đủ 5 biến môi trường trên Vercel.

5.3. Deploy lần đầu

1
Bấm Deploy.
2
Chờ trạng thái Ready.
3
Mở domain Vercel, ví dụ https://hong-nhung-bds.vercel.app.
4
Mở /admin để kiểm tra đăng nhập.

6. Sử dụng trang quản trị /admin

6.1. Đăng nhập

Vào:

https://ten-du-an.vercel.app/admin

Nhập mật khẩu đã đặt ở biến ADMIN_PASSWORD.

Hình 7: Màn hình đăng nhập CMS.

6.2. Sửa thông tin chung

Tab Thông tin chung dùng để sửa:

  • Tên thương hiệu.
  • Logo.
  • Số điện thoại, email, văn phòng.
  • Zalo, Facebook.
  • Danh sách loại hình BĐS trên menu.

6.3. Sửa dự án

1
Chọn tab Dự án.
2
Chọn tên dự án ở cột trái.
3
Sửa tên, slug, loại hình, địa chỉ, giá, ảnh, nội dung chi tiết.
4
Upload ảnh mới nếu cần. Ảnh sẽ được đưa vào assets/img/ khi lưu.
5
Bấm Lưu & xuất bản.

Hình 8: Tab Dự án trong CMS.

6.4. Sau khi bấm Lưu & xuất bản

  1. CMS gửi dữ liệu tới /api/save.
  2. API kiểm tra mật khẩu.
  3. API commit vào GitHub các file content/site.json, content/projects.json và ảnh mới nếu có.
  4. GitHub có commit mới.
  5. Vercel tự build lại.
  6. Website public cập nhật sau vài phút.

Khi thấy thông báo “Đã lưu lên GitHub. Vercel sẽ tự deploy lại trong ít phút”, nghĩa là CMS đã hoàn tất phần ghi dữ liệu.

7. Lỗi thường gặp và cách xử lý

LỗiNguyên nhân thường gặpCách xử lý
Không đăng nhập được /admin Sai ADMIN_PASSWORD hoặc chưa deploy lại sau khi đặt biến. Kiểm tra Vercel → Settings → Environment Variables. Sau đó redeploy.
Bấm lưu báo Unauthorized Mật khẩu gửi từ admin không khớp biến ADMIN_PASSWORD. Đăng xuất, đăng nhập lại bằng mật khẩu đúng.
Bấm lưu báo Missing GITHUB_TOKEN Chưa thêm biến môi trường trên Vercel. Thêm đủ 5 biến: ADMIN_PASSWORD, GITHUB_TOKEN, GITHUB_OWNER, GITHUB_REPO, GITHUB_BRANCH.
Bấm lưu báo GitHub API lỗi 404 Sai owner/repo/branch hoặc token không có quyền với repo. Kiểm tra lại GITHUB_OWNER, GITHUB_REPO, GITHUB_BRANCH. Tạo token chọn đúng repository.
Bấm lưu báo quyền Contents Token chưa có quyền Contents: Read and write. Tạo lại token fine-grained và cấp quyền Contents Read and write.
Vercel build lỗi JSON bị sửa sai cú pháp hoặc build command sai. Chạy local: node scripts/build.js. Trên Vercel dùng build command node scripts/build.js.
Upload ảnh trong admin lỗi Ảnh quá lớn hoặc mạng yếu. Nén ảnh dưới 4 MB rồi upload lại.
Website chưa đổi ngay Vercel đang build hoặc trình duyệt cache. Đợi 1-3 phút, mở Vercel Deployments kiểm tra trạng thái, sau đó refresh bằng Ctrl + F5.

Checklist bàn giao

  • GitHub repo đã có thư mục admin, api, content, scripts.
  • Vercel project deploy thành công.
  • Vercel có đủ 5 biến môi trường.
  • Mở được trang chủ.
  • Mở được /admin.
  • Sửa thử một dòng nhỏ, bấm lưu, thấy commit mới trong GitHub.
  • Vercel tự redeploy sau commit.
  • Website public hiển thị nội dung mới.
